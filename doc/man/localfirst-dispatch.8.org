#+TITLE: localfirst-dispatch

* NAME
localfirst-dispatch - delivery-time dispatcher for hybrid system/vpopmail domains

* SYNOPSIS
*| /var/qmail/bin/localfirst-dispatch* [ /HOST/ ]

* DESCRIPTION
*localfirst-dispatch* replaces a hybrid vpopmail domain's catch-all
=.qmail-default= (normally =| /home/vpopmail/bin/vdelivermail '' delete=). For
the recipient local-part (qmail's *$DEFAULT*, falling back to *$EXT*), if it is
a mail-eligible system identity (see *system-identity(8)*) the message is
re-injected to =LOCALPART@HOST= via qmail's *forward*, so qmail-send delivers it
to the system account with correct ownership. Otherwise the original vpopmail
catch-all (=vdelivermail '' delete=) is executed unchanged.

/HOST/ is the locals host domain to re-inject to. It is taken from the first
argument; if omitted, from =/var/qmail/control/me=. It must differ from the
vpopmail virtual domain so re-injection does not loop.

* NOTES
This program runs as the vpopmail user and deliberately does not touch any
recipient's =~/Maildir=: it cannot create a file owned by the target user. A
recipient's Maildir is created from =/etc/skel= at account creation; a
delivery-time fallback (running as the target user) creates it if absent.

* WIRING
In the vpopmail domain's =.qmail-default=:

: | /var/qmail/bin/localfirst-dispatch hoth.example.org

* SEE ALSO
*localfirst-rcptcheck(8)*, *system-identity(8)*, *forward(1)*

For the theory of operation, configuration, and debugging hints, see the
/dsmr-localfirst/ info manual (=info dsmr-localfirst=) or the stack
documentation website.
